## Technical Implementation

### Extraction Process

1. **Parse Header File**: Read `indigo_names.h` line by line
2. **Regex Matching**: Extract constants using pattern:

   ```rust
   r#"^#define\s+(?<name>\w+)_NAME\s+"(?<value>.+)"\s*$"#
   ```

3. **Store in BTreeMap**: Ensures alphabetical ordering
4. **Generate Rust Code**: Write to `src/constants.rs` with metadata
5. **Cargo Integration**: Automatic rerun when header changes

### Module Structure

Constants are accessible through the `libindigo::name` module:

```rust
use libindigo::name;

// Access constants
let conn = name::CONNECTION_PROPERTY;  // "CONNECTION"
let info = name::INFO_PROPERTY;        // "INFO"
```

The module is defined in [`src/lib.rs`](src/lib.rs:89-101):

```rust
pub mod name {
    include!("constants.rs");

    #[cfg(test)]
    mod tests {
        use crate::name;

        #[test]
        fn names() {
            assert_eq!(name::INFO_PROPERTY, "INFO");
        }
    }
}
```

## Build Integration

### When Constants Are Generated

- **Always**: If INDIGO submodule exists at `sys/externals/indigo`
- **Fallback**: Uses existing `src/constants.rs` if submodule missing
- **Clean Build**: `cargo clean && cargo build` forces regeneration

### Build Script Logic

```rust
fn main() -> std::io::Result<()> {
    let submodule = Path::new(INDIGO_GIT_SUBMODULE);

    // Always try to extract constants if submodule exists
    if submodule.exists() {
        if let Err(e) = extract_indigo_constants(&submodule) {
            eprintln!("Warning: Could not extract INDIGO constants: {}", e);
            eprintln!("Continuing with existing constants.rs");
        }
    }

    // ... FFI bindings generation for FFI features ...
}
```
